`### **`pycemrg-model-creation` API Reference**

#### **1. Overview**

The `pycemrg-model-creation` library provides a high-level, Pythonic framework for executing complex cardiac modeling workflows, such as UVC surface extraction. It is designed with a layered architecture to provide both convenience for common tasks and flexibility for advanced users.

**The typical user workflow is:**
1.  Initialize low-level dependencies (`pycemrg.data.LabelManager` and a tool wrapper like `MeshtoolWrapper`).
2.  Use the `PathContractBuilder` to generate all necessary input/output paths for a workflow from a few high-level directories.
3.  Instantiate the main `SurfaceLogic` class with the dependencies.
4.  Call a high-level workflow method (e.g., `run_ventricular_extraction()`) on the `SurfaceLogic` instance, passing it the paths generated by the builder.

---

#### **2. Set your project paths easily: `PathContractBuilder`**

This is the recommended starting point for most workflows. It simplifies the setup process by generating the complex path contracts required by the `SurfaceLogic` engine.

**Entry Point:** `pycemrg_model_creation.logic.PathContractBuilder`

**Initialization:**
```python
from pathlib import Path
from pycemrg_model_creation.logic import PathContractBuilder

# Provide a single root directory for all outputs
output_dir = Path("/path/to/patient_01/uvc_surfaces")
path_builder = PathContractBuilder(output_dir=output_dir)
```

**Methods:**

*   **`build_all(mesh_base_path: Path, blank_files_dir: Path) -> UVCSurfaceExtractionPaths`**:
    *   **Purpose:** The main convenience method. It constructs the master path contract for the entire UVC surface extraction workflow.
    *   **Args:**
        *   `mesh_base_path`: The base path to the input four-chamber mesh (e.g., `Path("myocardium")`).
        *   `blank_files_dir`: Directory containing template/blank files for atrial apex/septum points.
    *   **Returns:** A fully populated `UVCSurfaceExtractionPaths` dataclass instance, ready to be used by `SurfaceLogic`.

*   **Granular Builders**: The class also provides more specific builder methods (`build_ventricular_paths`, `build_atrial_paths`, etc.) for users who only need to run parts of the full workflow.

---

#### **3. The "Engine": `SurfaceLogic` Class**

This class is the stateless orchestrator for the scientific workflow of extracting cardiac surfaces. It contains the "how-to" logic for the entire process.

**Entry Point:** `pycemrg_model_creation.logic.SurfaceLogic`

**Initialization:**
```python
from pycemrg.data import LabelManager
from pycemrg_model_creation.tools import MeshtoolWrapper
from pycemrg_model_creation.logic import SurfaceLogic

# 1. Initialize dependencies first
# Assumes 'meshtool' is in the system PATH
meshtool = MeshtoolWrapper.from_system_path() 
# Assumes a 'labels.yaml' file exists for the project
label_manager = LabelManager("/path/to/project/labels.yaml")

# 2. Instantiate the logic class
surface_logic = SurfaceLogic(meshtool, label_manager)
```

**Primary Workflow Method:**

*   **`run_ventricular_extraction(paths: VentricularSurfacePaths) -> None`**:
    *   **Purpose:** Executes the complete, sequential workflow to extract all necessary ventricular surfaces (base, epi, endo, septum) and generate their corresponding `.vtx` files.
    *   **Args:**
        *   `paths`: A `VentricularSurfacePaths` dataclass instance, typically generated by `PathContractBuilder`.
    *   **Side Effects:** Creates numerous surface and `.vtx` files in the directories specified within the `paths` contract.

*   **Granular Methods**: The class exposes the individual steps of the workflow (`extract_ventricular_base`, `extract_septum`, etc.) as public methods for advanced users who may need to customize the process.

**Unimplemented API**: Methods related to atrial and submesh extraction are present but not yet implemented.

---

#### **4. The "Power Tools": `MeshtoolWrapper` & `CarpWrapper`**

These classes provide low-level, direct Python wrappers for the `meshtool` and `openCARP` command-line utilities. They are used internally by `SurfaceLogic` but are also available for advanced users needing to perform one-off tasks.

**Entry Point:** `pycemrg_model_creation.tools.MeshtoolWrapper`
**Entry Point:** `pycemrg_model_creation.tools.CarpWrapper`

**Initialization:**

*   **For `meshtool` (standalone):**
    ```python
    from pycemrg_model_creation.tools import MeshtoolWrapper
    meshtool = MeshtoolWrapper.from_system_path()
    ```

*   **For tools within a CARPentry environment:**
    ```python
    from pycemrg.system import CommandRunner, CarpRunner
    from pycemrg_model_creation.tools import MeshtoolWrapper, CarpWrapper
    
    runner = CommandRunner()
    carp_runner = CarpRunner(runner, carp_config_path="/path/to/config.sh")

    meshtool = MeshtoolWrapper.from_carp_runner(carp_runner)
    carp_tools = CarpWrapper(carp_runner)
    ```

**Methods:**
Each class provides Pythonic methods that map directly to the subcommands of the tool (e.g., `meshtool.convert(...)`, `meshtool.extract_surface(...)`, `carp_tools.gl_rule_fibres(...)`). Refer to the class docstrings for detailed arguments.

---

#### **5. Full Usage Example**

This example ties everything together, demonstrating the intended use of the library from an orchestrator script like `ProjectScar`.

```python
from pathlib import Path
from pycemrg.data import LabelManager
from pycemrg_model_creation.logic import PathContractBuilder, SurfaceLogic
from pycemrg_model_creation.tools import MeshtoolWrapper

# --- 1. Define High-Level Paths and Config ---
project_dir = Path("/data/patient_01")
output_dir = project_dir / "uvc_surfaces"
mesh_base_path = project_dir / "meshing/myocardium"
labels_config = project_dir / "config/labels.yaml"

# --- 2. Initialize Dependencies ---
meshtool = MeshtoolWrapper.from_system_path()
label_manager = LabelManager(labels_config)

# --- 3. Generate the Path Contract ---
# The builder handles creation of all subdirectories and file paths.
path_builder = PathContractBuilder(output_dir=output_dir)
# For this example, we only need the ventricular part of the master contract.
all_paths = path_builder.build_all(mesh_base_path, blank_files_dir=project_dir / "config")
ventricular_paths = all_paths.ventricular

# --- 4. Initialize and Run the Logic Engine ---
surface_logic = SurfaceLogic(meshtool, label_manager)
surface_logic.run_ventricular_extraction(paths=ventricular_paths)

print("Ventricular surface extraction complete.")
```
